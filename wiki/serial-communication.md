# 串口通信

## 什么是串口？

串口（UART）是一种让两个设备通信的方式，一次发一个 bit。ESP32 通过 USB 转串口芯片 (CP2102) 连接到电脑。

## 波特率 (Baud Rate)

**你决定的。** `Serial.begin(115200)` 就是你在代码里设置的通信速度。

### 常用波特率

| 波特率 | 适用场景 |
|--------|---------|
| 9600 | 标准、稳定，传输距离远 |
| 57600 | 一般情况 |
| **115200** | **ESP32 最常用** — 速度和稳定性平衡点 |
| 921600 | ESP32 支持，距离短且可能丢数据 |

### 关键规则
- 发送方和接收方必须用**相同的波特率**
- `platformio.ini` 的 `monitor_speed` 必须与 `Serial.begin()` 一致
- 波特率越高传输越快，但越容易受干扰

### 为什么 115200？
不是芯片决定的，也不是电脑决定的——是 **工程惯例 + 你自己选择**。115200 是 UART 协议族中的一个标准速率，够快又稳定，所以大多数教程和库都默认用它。

## Serial Monitor 连接时序

- ESP32 通过 USB 连接到电脑的 `/dev/cu.usbserial-0001`
- 打开串口监视器时，DTR 信号会导致 ESP32 复位
- 此时 ESP32 重新运行 `setup()`，但监视器需要时间建立连接
- 所以 `setup()` 里的打印有时会看不到
- 加 `delay(1000)` 可以让监视器有足够时间连上
- ESP32 启动时 ROM bootloader 输出乱码 → 正常现象，不理即可
